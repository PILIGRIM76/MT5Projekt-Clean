# Инструкция по переобучению моделей

## Проблема
Модели BITCOIN и USDJPY несовместимы с текущим набором признаков:
- Старые LSTM модели ожидают 24 признака (с дублирующими KG признаками)
- Система предоставляет 20 признаков (без KG признаков)
- Результат: `RuntimeError: input.size(-1) must be equal to input_size. Expected 24, got 20`

## Решение

### Метод 1: Автоматическое переобучение (РЕКОМЕНДУЕТСЯ)

Используйте скрипт `retrain_symbols_simple.py` для удаления старых моделей:

```cmd
python retrain_symbols_simple.py BITCOIN USDJPY
```

Скрипт:
1. Подключится к базе данных `F:\Enjen\trading_system.db`
2. Удалит старые модели из таблицы `trained_models`
3. Покажет количество удаленных моделей

**Результат для BITCOIN и USDJPY**: Удалено 5387 моделей ✅

После запуска скрипта:
1. Запустите торговую систему: `python main_pyside.py`
2. Дождитесь выполнения R&D цикла (каждые 5 минут)
3. Новые модели будут использовать правильный набор признаков (20 без KG)
4. Проверьте логи на наличие сообщений `[R&D]` для отслеживания прогресса

### Метод 2: Ручное удаление из базы данных

Если у вас есть доступ к базе данных SQLite:

1. Откройте базу данных: `F:\Enjen\trading_system.db`
2. Выполните SQL запрос:
```sql
DELETE FROM trained_models WHERE symbol IN ('BITCOIN', 'USDJPY');
```
3. Запустите систему - R&D цикл переобучит модели автоматически

### Метод 3: Ожидание автоматического переобучения

Система автоматически переобучает модели:
- Каждую неделю (по расписанию)
- При обнаружении дрейфа модели
- При низкой производительности модели

Просто дождитесь следующего цикла переобучения (может занять до недели).

## Частые вопросы

**Q: Сколько времени занимает переобучение?**
A: R&D цикл выполняется каждые 5 минут и занимает 6-12 секунд на символ.

**Q: Потеряю ли я историю торговли?**
A: Нет, удаляются только модели. История сделок, позиции и настройки сохраняются.

**Q: Можно ли переобучить все символы сразу?**
A: Да, просто перечислите их: `python retrain_symbols.py BITCOIN USDJPY XAUUSD EURUSD`

**Q: Что делать, если скрипт выдает ошибку?**
A: Проверьте:
- Путь к базе данных: `F:\Enjen\trading_system.db`
- Наличие файла базы данных
- Права доступа к файлу базы данных
- Таблица должна называться `trained_models`
