# 🔧 Исправление обучения (R&D)

## 📋 Проблема

**Симптомы:**
- ❌ Обучение не запускается
- ❌ Нет логов о начале R&D цикла
- ❌ `latest_full_ranked_list` пуст

**Причины:**
1. **Блокировка analysis_lock** - проверка внутри lock блокирует торговлю
2. **Нет данных** - `latest_full_ranked_list` не заполняется
3. **Слишком долгая задержка** - 60 сек ожидания между циклами

---

## ✅ Решение

### **1. Исправлен `_training_loop`**

**Файл:** `src/core/trading_system.py`
**Строки:** 1024-1063

**Изменения:**
```python
# БЫЛО:
self.stop_event.wait(60)  # Ждал 60 сек перед стартом

# СТАЛО:
logger.info("[R&D] Ожидание 120 сек для накопления данных...")
self.stop_event.wait(120)  # Ждем 2 минуты для накопления данных
```

**Дополнительно:**
- ✅ Добавлена проверка `latest_full_ranked_list`
- ✅ Добавлено детальное логирование
- ✅ Проверка `is_heavy_init_complete`

### **2. Исправлен `_continuous_training_cycle`**

**Проблема:** Проверка `latest_full_ranked_list` внутри `analysis_lock`

**Решение:** Вынести проверку за пределы lock

```python
# ПРОВЕРКА ДО LOCK
if not self.latest_full_ranked_list:
    logger.warning("[R&D] Список пуст. Запуск сбора данных...")
    # Принудительный сбор данных

# ТОЛЬКО ПОТОМ БЛОКИРОВКА
with self.analysis_lock:
    # Обработка данных
```

## ⚠️ Важные замечания

### **1. Время обучения**

Обучение 3 моделей занимает ~30-60 секунд

### **2. Требования к данным**

Минимум 1000 баров для обучения:
```python
if len(df_full) < 1000:
    logger.warning("[R&D] Недостаточно данных")
```

### **3. Блокировка**

`training_lock` предотвращает одновременное обучение:
```python
if not self.training_lock.acquire(blocking=False):
    logger.warning("[R&D] Обучение уже идет")
    return
```
