# Как использовать промт для улучшения системы

## Быстрый старт

### Вариант 1: Использование с Kiro (текущий AI)

Просто скажите:

```
Прочитай файл SYSTEM_IMPROVEMENT_PROMPT.md и начни внедрять изменения по приоритету.
Начни с Priority 1 (CRITICAL).
```

### Вариант 2: Использование с другим AI (Claude, GPT-4, etc.)

1. Откройте файл `SYSTEM_IMPROVEMENT_PROMPT.md`
2. Скопируйте весь текст
3. Вставьте в начало разговора с AI
4. Добавьте: "Начни внедрение с Priority 1 (CRITICAL). Работай пошагово."

---

## Пошаговый план внедрения

### ШАГ 1: Обновление конфигурации (30 минут)

**Команда для AI:**
```
Обнови файл configs/settings.json согласно разделу "1. RISK MANAGEMENT OVERHAUL".
Покажи мне diff изменений перед применением.
```

**Ожидаемый результат:**
- RISK_PERCENTAGE: 2.0 → 0.5
- MAX_OPEN_POSITIONS: 18 → 5
- STOP_LOSS_ATR_MULTIPLIER: 2.5 → 3.5
- MAX_DAILY_DRAWDOWN_PERCENT: 10.0 → 5.0

### ШАГ 2: Добавление валидации моделей (1 час)

**Команда для AI:**
```
Добавь функцию _validate_model_metrics() в src/core/trading_system.py
согласно разделу "2. MODEL TRAINING & VALIDATION OVERHAUL".
Интегрируй её в _run_champion_contest().
```

**Ожидаемый результат:**
- Новая функция `_validate_model_metrics()`
- Модели с Profit Factor < 1.5 будут отклоняться
- Модели с Win Rate < 40% будут отклоняться

### ШАГ 3: Создание Safety Monitor (1 час)

**Команда для AI:**
```
Создай файл src/core/safety_monitor.py согласно разделу "4. MONITORING & EMERGENCY STOPS".
Интегрируй SafetyMonitor в TradingSystem.
```

**Ожидаемый результат:**
- Новый файл `src/core/safety_monitor.py`
- Автоматическая остановка при просадке > 3%
- Автоматическая остановка при 5 убыточных сделках подряд

### ШАГ 4: Обновление параметров обучения (30 минут)

**Команда для AI:**
```
Обнови параметры обучения в configs/settings.json:
- TRAINING_DATA_POINTS: 2000 → 10000
- TRAINING_INTERVAL_SECONDS: 300 → 86400
- LSTM epochs: 20 → 50
```

### ШАГ 5: Walk-Forward Validation (2 часа)

**Команда для AI:**
```
Реализуй функцию _walk_forward_validation() в src/core/trading_system.py
согласно разделу "2. MODEL TRAINING & VALIDATION OVERHAUL".
```

**Ожидаемый результат:**
- Модели тестируются на 5 out-of-sample периодах
- Модель принимается только если прибыльна на ВСЕХ периодах

### ШАГ 6: Улучшение сигналов (1 час)

**Команда для AI:**
```
1. Обнови пороги сигналов в configs/settings.json (раздел 3)
2. Добавь _filter_low_quality_signals() в src/core/services/signal_service.py
3. Исправь дубликаты признаков в src/ml/feature_engineer.py
```

## Проверка результатов

После каждого шага запускайте:

```bash
# Проверка синтаксиса
python -m py_compile src/core/trading_system.py
python -m py_compile src/core/safety_monitor.py
python -m py_compile src/core/services/signal_service.py

# Запуск системы в режиме наблюдателя
python main_pyside.py
```

**Что проверять:**
1. Система запускается без ошибок
2. В логах появляются сообщения "[SAFETY] Monitoring initialized"
3. R&D цикл запускается раз в день (не каждые 5 минут)
4. Модели с плохими метриками отклоняются (смотрите "MODEL REJECTED" в логах)

## Критические моменты

### ⚠️ ВАЖНО: Перед запуском на реальном счёте

1. **Протестируйте на демо минимум 30 дней**
2. **Убедитесь, что все модели прошли валидацию**
3. **Проверьте, что Safety Monitor работает** (искусственно создайте убыток и проверьте остановку)
4. **Начните с минимального депозита** ($1000-2000)

### ⚠️ Что делать, если система отклоняет все модели

Это нормально! Это означает, что текущие модели действительно убыточны.

**Решение:**
1. Увеличьте TRAINING_DATA_POINTS до 20000 (больше данных = лучше модели)
2. Подождите 1-2 недели накопления новых данных
3. Попробуйте другие символы (EURUSD обычно стабильнее BITCOIN)
4. Рассмотрите упрощение стратегии (меньше признаков, проще модели)

## Мониторинг после внедрения

### Ежедневно проверяйте:
- [ ] Количество отклонённых моделей (должно быть 70-90% в первые недели)
- [ ] Количество принятых моделей (должно быть 1-3 модели с хорошими метриками)
- [ ] Дневная просадка (должна быть < 3%)
- [ ] Количество открытых позиций (должно быть ≤ 5)

### Еженедельно проверяйте:
- [ ] Profit Factor всех активных моделей (должен быть > 1.5)
- [ ] Win Rate всех активных моделей (должен быть > 40%)
- [ ] Общая прибыльность системы
- [ ] Логи на наличие ошибок или предупреждений

### Ежемесячно проверяйте:
- [ ] Общий Sharpe Ratio (цель > 1.0)
- [ ] Максимальная просадка (цель < 10%)
- [ ] Стабильность системы (отсутствие крашей)
- [ ] Качество исполнения ордеров (проскальзывание)

## Ожидаемые результаты

### Первая неделя:
- 90% моделей будут отклонены (это хорошо!)
- Система будет торговать редко или не торговать вообще
- Это нормально - система ждёт качественных сигналов

### Первый месяц:
- 2-3 модели пройдут валидацию
- Система начнёт торговать 1-2 раза в день
- Profit Factor должен быть > 1.2
- Win Rate должен быть > 35%

### Через 3 месяца:
- 5-7 моделей активны
- Система торгует 3-5 раз в день
- Profit Factor > 1.5
- Win Rate > 40%
- Sharpe Ratio > 1.0
- Готовность к переходу на реальный счёт

## Поддержка и вопросы

Если что-то пошло не так:

1. **Проверьте логи:** `logs/` директория
2. **Проверьте конфигурацию:** `configs/settings.json`
3. **Откатите изменения:** используйте git для отката к предыдущей версии
4. **Спросите AI:** "Что пошло не так? Вот ошибка из логов: [вставьте ошибку]"
