# 🎯 СИСТЕМА АВТОМАТИЧЕСКОГО УПРАВЛЕНИЯ СИМВОЛАМИ

## 📋 Описание

Система теперь **автоматически анализирует производительность символов** и принимает решения об их исключении/включении на основе объективных метрик.

---

## 🔧 Как это работает

### 1️⃣ **Мониторинг производительности**

Фоновый поток `_symbol_performance_monitor_loop` постоянно отслеживает:
- Общую прибыль по каждому символу
- Profit Factor (PF)
- Win Rate (WR)
- Количество сделок

### 2️⃣ **Критерии исключения символов**

Символ исключается из торговли, если выполняются **ЛЮБОЕ** из условий:

| Критерий | Порог | Пример |
|----------|-------|--------|
| **Общий убыток** | < -$500 | -$800 → Исключить |
| **Profit Factor** | < 0.8 | 0.65 → Исключить |
| **Win Rate** | < 40% | 35% → Исключить |
| **Мин. сделок** | ≥ 10 | 15 сделок для статистики |

### 3️⃣ **Критерии включения символов**

Исключенный символ включается обратно, если:

| Критерий | Порог | Пример |
|----------|-------|--------|
| **Profit Factor** | > 1.2 | 1.5 → Включить |
| **Win Rate** | > 55% | 60% → Включить |
| **Общая прибыль** | > $0 | +$250 → Включить |
| **Мин. сделок** | ≥ 5 | 8 сделок для статистики |

## ⚙️ Настройка порогов

Измените параметры в методе `get_symbols_for_auto_exclusion`:

```python
candidates_for_exclusion = self.db_manager.get_symbols_for_auto_exclusion(
    min_trades=10,                # Мин. сделок для анализа
    max_loss_threshold=-500.0,    # Макс. убыток ($)
    profit_factor_threshold=0.8,  # Мин. PF
    win_rate_threshold=0.40       # Мин. WR
)
```

Для включения:

```python
candidates_for_inclusion = self.db_manager.get_symbols_for_auto_inclusion(
    excluded_symbols=list(current_excluded),
    min_trades=5,                 # Мин. сделок для анализа
    profit_factor_threshold=1.2,  # Мин. PF
    win_rate_threshold=0.55       # Мин. WR
)
```
