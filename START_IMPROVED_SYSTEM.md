## ⚠️ ЧТО ОЖИДАТЬ

### НОРМАЛЬНО (не паникуй!):

1. **Система почти не торгует**
   - Это ХОРОШО! Она ждёт качественных сигналов
   - Может пройти несколько дней без сделок

2. **В логах много "MODEL REJECTED"**
   - Это ОТЛИЧНО! Валидация работает
   - Старые убыточные модели отклоняются

3. **В логах много "Signal rejected"**
   - Это ПРАВИЛЬНО! Фильтры работают
   - Слабые сигналы отклоняются

4. **R&D цикл запускается раз в день**
   - Раньше было каждые 5 минут
   - Теперь система не переобучается

### НЕ НОРМАЛЬНО (требует внимания):

1. **Система крашится**
   - Проверь логи: `logs\trading_system.log`
   - Найди последнюю ошибку (ERROR или CRITICAL)

2. **Safety Monitor не инициализируется**
   - Проверь подключение к MT5
   - Проверь логи на ошибки

3. **Открывается >5 позиций**
   - Конфигурация не применилась
   - Перезапусти систему

4. **Модели принимаются с PF<1.5**
   - Валидация не работает
   - Проверь код в `trading_system.py`

## 🔧 УСТРАНЕНИЕ ПРОБЛЕМ

### Проблема: Система не запускается

**Решение:**
```powershell
# Проверь последние ошибки
Select-String -Path logs\trading_system.log -Pattern "ERROR|CRITICAL" | Select-Object -Last 10

# Если ошибка в импорте safety_monitor:
python -m py_compile src\core\safety_monitor.py

# Если ошибка в конфигурации:
# Проверь configs/settings.json на валидность JSON
```

### Проблема: Safety Monitor не работает

**Решение:**
```powershell
# Проверь, что файл создан
Test-Path src\core\safety_monitor.py

# Проверь логи на ошибки инициализации
Select-String -Path logs\trading_system.log -Pattern "SAFETY.*error|SAFETY.*ошибка" -CaseSensitive:$false
```

3. Новые модели будут лучше
