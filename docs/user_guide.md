## 1. Начало работы

### 1.1 Первый запуск

1. **Установите MetaTrader 5** и откройте демо-счёт (рекомендуется для начала).
2. **Запустите** `main_pyside.py` или `.exe` файл.
3. Перейдите в **Настройки** (кнопка ⚙️) → вкладка **Подключение MT5**.

### 1.2 Режимы работы

| Режим | Описание | Когда использовать |
|-------|----------|-------------------|
| **Paper Trading** | Симуляция без реальных денег | Обучение, тестирование стратегий |
| **Demo Trading** | Торговля на демо-счёте MT5 | Проверка системы в реальных условиях |
| **Live Trading** | Реальная торговля | Только после thorough тестирования! |

### 1.3 Интерфейс

Главное окно содержит вкладки:
- **🏠 Главная** — обзор баланса, позиций, статуса систем
- **📊 Графики** — графики цен и индикаторов
- **📈 Торговля** — управление позициями и ордерами
- **🧪 Бэктестинг** — тестирование стратегий на истории
- **⚙️ Настройки** — все параметры системы
- **📝 Логи** — журнал событий и ошибок

## 3. Настройка торговых символов

### 3.1 White-list символов

**Путь:** Настройки → Подключение MT5 → Символы

Система торгует только символами из этого списка. Рекомендуемый набор:

```json
["EURUSD", "GBPUSD", "USDJPY", "AUDUSD", "USDCAD",
 "NZDUSD", "USDCHF", "EURGBP", "EURJPY", "GBPJPY",
 "XAUUSD", "XAGUSD", "BTCUSD", "US500", "US100",
 "DE40", "UK100", "SUGAR"]
```

### 3.2 Исключение символов

**Путь:** Настройки → Подключение MT5 → Исключённые символы

Символы, которые система будет **игнорировать** даже если они в white-list:

```json
["USDRUB", "EURRUB"]
```

### 3.3 Рекомендуемое количество

| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| `TOP_N_SYMBOLS` | 15-20 | Макс. символов для сканирования |
| `MAX_OPEN_POSITIONS` | 8-12 | Макс. одновременных позиций |

> ⚠️ **Не ставьте слишком много** — это снижает качество сигналов и увеличивает риск.

## 4. Управление стратегиями

### 4.1 Доступные стратегии

| Стратегия | Тип | Когда работает |
|-----------|-----|----------------|
| **Breakout** | Пробой уровней | При сильных движениях и пробоях |
| **Mean Reversion** | Возврат к среднему | Во флэте, при отклонении от средней |
| **MA Crossover** | Пересечение скользящих | В трендовых условиях |
| **Adaptive** | Адаптивная | Автоматически подстраивается |

### 4.2 Вес стратегий

**Путь:** Настройки → Торговля → Веса стратегий

Система использует взвешенный консенсус:

```json
{
  "breakout": 0.35,
  "mean_reversion": 0.25,
  "ma_crossover": 0.25,
  "adaptive": 0.15
}
```

> 💡 **Совет:** Увеличивайте вес стратегии, которая хорошо работает на текущем рынке.

### 4.3 Параметры стратегий

#### Breakout Strategy
| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `lookback_period` | 20 | Период для поиска уровней |
| `atr_multiplier` | 1.5 | Множитель ATR для стопа |
| `volume_threshold` | 1.2 | Порог объёма для подтверждения |

#### Mean Reversion
| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `window` | 163 | Период скользящей средней |
| `entry_threshold` | 2.0 | Порог отклонения для входа |
| `exit_threshold` | 0.5 | Порог возврата для выхода |

#### MA Crossover
| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `fast_period` | 18 | Быстрая скользящая |
| `slow_period` | 56 | Медленная скользящая |

## 5. Риск-менеджмент

### 5.1 Основные параметры риска

**Путь:** Настройки → Торговля → Риск-менеджмент

| Параметр | По умолчанию | Описание | Рекомендуемый диапазон |
|----------|-------------|----------|----------------------|
| `RISK_PERCENTAGE` | 2.0 | % баланса на сделку | 1-5% |
| `STOP_LOSS_ATR_MULTIPLIER` | 2.0 | Стоп-лосс в ATR | 1.5-3.0 |
| `TAKE_PROFIT_ATR_MULTIPLIER` | 3.0 | Тейк-профит в ATR | 2.0-4.0 |
| `MAX_DAILY_DRAWDOWN_PERCENT` | 5.0 | Макс. дневная просадка (%) | 3-10% |
| `MAX_DRAWDOWN_PERCENT` | 15.0 | Макс. общая просадка (%) | 10-20% |

### 5.2 Корреляционный анализ

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `CORRELATION_THRESHOLD` | 0.8 | Макс. корреляция между позициями |

> ⚠️ Если две позиции коррелируют > 0.8, система отклонит вторую.

### 5.3 Trailing Stop

**Путь:** Настройки → Торговля → Trailing Stop

| Параметр | Описание |
|----------|----------|
| `enabled` | Включить trailing stop |
| `atr_multiplier` | Множитель ATR для трейлинга |
| `min_profit` | Мин. прибыль для активации |

### 5.4 Pre-Mortem анализ

Система выполняет **GARCH Monte Carlo** симуляцию перед открытием позиции для оценки риска катастрофических потерь.

**Путь:** Настройки → Торговля → Риск → `pre_mortem.enabled`

## 8. Оркестратор (RL)

### 8.1 Что это такое

Оркестратор — это **RL агент (PPO)**, который:
- Распределяет капитал между стратегиями
- Адаптируется к рыночному режиму (тренд/флэт/волатильность)
- Самостоятельно обучается на накопленном опыте

### 8.2 Настройка

**Путь:** Настройки → R&D (AI) → Оркестратор

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `enabled` | true | Включить оркестратор |
| `model_path` | `F:/Enjen/orchestrator` | Путь к модели |
| `decision_interval` | 14400 | Интервал решений (4 часа, в секундах) |
| `training_interval` | 604800 | Интервал переобучения (7 дней) |

### 8.3 Рекомендации

- ✅ Включайте **после** того, как система поработает хотя бы неделю
- ✅ Оркестратору нужны данные для обучения — не включайте на пустой системе
- ✅ Путь к модели должен быть на **быстром диске (SSD)**

## 9. R&D цикл и генетическое программирование

### 9.1 Автоматический R&D

Система **самостоятельно создаёт и тестирует** новые стратегии:

1. **Генерация** — GP создаёт новые стратегии
2. **Инкубация** — 30 дней тестирования
3. **Оценка** — успешные переходят в "live"
4. **Увольнение** — неудачные деактивируются

### 9.2 Настройка R&D цикла

**Путь:** Настройки → R&D (AI) → R&D цикл

| Параметр | Описание | Рекомендуемое значение |
|----------|----------|----------------------|
| `enabled` | Включить R&D | `true` |
| `cycle_interval_hours` | Интервал цикла | 168 (1 неделя) |
| `incubation_period_days` | Срок инкубации | 30 |
| `min_sharpe_ratio` | Мин. Шарп для принятия | 1.0 |
| `max_drawdown_threshold` | Макс. просадка для отклонения | 15% |

2. Проверьте **путь к terminal64.exe**
3. Проверьте **логин/пароль/сервер**
4. Попробуйте **перезапустить** MT5

### ❓ Слишком много сделок

1. Уменьшите `TOP_N_SYMBOLS` (10-15)
2. Уменьшите `MAX_OPEN_POSITIONS` (5-8)
3. Увеличьте `RISK_PERCENTAGE` (меньше сделок при большем риске на сделку)
4. Проверьте **веса стратегий** — уменьшите, если все 1.0

### ❓ Модели не обучаются

1. Проверьте, что достаточно **данных** (мин. 1000 точек)
2. Увеличьте `TRAINING_DATA_POINTS` до 5000
3. Проверьте **GPU/CPU** — PyTorch может требовать ресурсы
4. Посмотрите **логи** на наличие ошибок

### ❓ Как отключить AI/ML?

1. Перейдите в **Настройки → R&D (AI)**
2. Отключите `auto_retraining.enabled`
3. Установите веса AI в консенсусе в `0`
4. Увеличьте веса классических стратегий

### ❓ Как увеличить скорость работы?

1. Уменьшите `GP_POPULATION_SIZE` и `GP_GENERATIONS`
2. Увеличьте `TRAINING_INTERVAL_SECONDS`
3. Отключите неиспользуемые стратегии
4. Используйте **LightGBM** вместо LSTM/Transformer

### ❓ Как сделать бэкап?

1. Скопируйте папку `F:/Enjen/database/`
2. Скопируйте `configs/settings.json`
3. Сохраните API ключи (они в keyring ОС)
